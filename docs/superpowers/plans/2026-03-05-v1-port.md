# Agentstrove v1 Port Implementation Plan

> **Status: COMPLETE** — All chunks implemented and dogfood-tested against hosted ClickHouse Cloud (850+ sessions synced successfully).

**Goal:** Port the v0 prototype to v1 with ClickHouse storage backend, shipping sync daemon, conversation browser, FTS search, git linking, and secret masking.

**Architecture:** Go backend with stdlib `net/http`, ClickHouse for storage (ReplacingMergeTree for dedup, built-in FTS), Svelte 5 frontend. The sync engine reads from agentsview SQLite, masks secrets, and writes to ClickHouse. The API server reads from ClickHouse and serves the SPA.

**Tech Stack:** Go 1.24+, ClickHouse (clickhouse-go/v2), SQLite (go-sqlite3), Svelte 5, Vite, TypeScript, testify

**Spec:** `docs/superpowers/specs/2026-03-05-v1-port-design.md`
**Roadmap:** `docs/roadmap.md`
**Testing:** `docs/testing.md`
**v0 Reference:** `../agentstrove/` (working prototype)

ClickHouse FTS approach: use `hasToken()` or `multiSearchAny()` for token matching on `messages.content`. For snippet extraction, use `extract()` or substring around match position.

The Search method should:
1. Query messages table for content matching the search query
2. Join with sessions to get session metadata
3. Extract a snippet (~200 chars) around the match
4. Calculate highlight positions within the snippet
5. Apply filters (user_id, project_id, agent_type, date range)
6. Limit results
7. Return SearchPage with total count

If ClickHouse's built-in FTS proves difficult, a simpler approach is `positionCaseInsensitive(content, query)` for basic matching. Start simple and iterate.
